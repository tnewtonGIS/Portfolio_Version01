Capstone

Automating field data and reporting for Ghana Disease Mitigation

Problem

Ghana, Africa, located near the equator, struggles to control neglected tropical diseases (NTD) in rural areas. USAID teamed up with the Ghana Ministry of Health to enhance the country's treatment program by establishing a yearly mass drug administration (MDA) since 2006. Since the initial distribution, tracking has been through paper records, leading to accountability issues in the number of people treated. FHI360, a Durham, North Carolina company, was awarded USAID and Gates Foundation grants to design, train, and implement a pilot program for electronic tracking. FHI360 requested help automating a system that could aggregate the field data and display it in a web application for the Ghana health workers to track daily progress.

Analysis Procedures

The raw data would be delivered nightly as a CSV file from the GPS "Tally Counter" device. The data required identifying key attributes representing a new contact point (home or road intersection). Each subsequent person after that would represent multiple people at that location, as the GPS device would mark each person with a series of attributes. The data would require aggregating each person to an area, collecting mean GPS coordinates for the location, create new tables to represent various statistics in a web application. Due to COVID restrictions in Ghana, the testing was delayed, so I designed a script to automate the incoming random data of the collectors. The following script cleaned all the data and pushed it into tables, while the final script pushed those CSV files to a PostgreSQL server. Scripting is written with R, and Esri model builder created geoprocesing tools embedded in the Esri web app builder (WAB) that would allow Ghana health officials to export the information as shape files for future needs. A secondary internal WAB displays metrics for investors and USAID to monitor progress.

Results

Beta testing ensures the data will be written from start to finish and ready for field collectors to begin uploading collection data. The web application does multiple operations, which will help distribute and target more households for treatment. Queries allow viewing of medication refusals or people not found. They can view daily results or overall results. Pill icons help keep track of total pharmacy numbers. Age, gender, and refusal locations help workers create target campaigns to address community concerns. A pattern analysis tool, not discussed yet, finds the hot spot clusters of refusals, displays the directional tendency of households, and creates a median location valid for support cells in distribution. The project met all the end user requirements with additional relevant and beneficial features to the project partners.

Database

ArcGIS Server and Geospatial Tools for the NPS

Problem

Stand up an Esri Server and create an Enterprise Database (SDE) for the National Park Service (NPS) to allow versioning for historic program changes to infrastructure and natural resources. Once complete, create a geospatial tool and web app (WAB) to display impacts on resources quickly.

Analysis Procedures

GIS and IT departments working in tandem to use Windows Server 2016, deploying security certificates and software ArcGIS Desktop, ArcServer Manager, Web Adapter, and PostgreSQL. Then, design the relational database structure using an Entity-Relationship (ER) Diagram, Lucid Charts, Unified Modeling Language (UML), and Parse-Dia-SQL. Once the infrastructure is complete, use SQL to read in CSV files to populate tables to create feature classes. Post-data collection, a geoprocessing tool designed using the Esri model builder, provided the WAB user functionality.

Results

The design allowed me to create notional datasets of fish species that could be routinely updated based on biologists' field collection studies. The scientist then can report time series changes using historical data sets. Scientists could quickly see if any changes to the release of species would be impacted by surrounding infrastructure.

Modeling

Sea Rise Analysis in America Samoa

Problem

The Pacific Ocean is home to dozens of low-lying countries. America Samoa was chosen for its access to available NOAA LIDAR datasets. The purpose was to understand the displacement of people from their homes and the potential loss of farmland due to salination.

Analysis Procedures

I focused on the Pago Pago Bay area to see the impact on the seaport, which is how the island receives most of its food source. Census data at block and tract levels were aggregated to buildings based on sqft to estimate the number of people per building. NAIP rasters were used in farmland designation. Using Lidar and fishnet grids, a tension spline modeled the sea height within the bay. A 15mm yearly sea rise published by the IPCC was used for decade encroachment.

Results

From 2021 to 2100, sea rise will not affect America Somoa. There are 58 buildings (1.97% of structures) in Pago Pago Bay affected, as most are built at higher elevations. Impacts on structures may be reduced with seawalls or increased dredging. Farming land in Pago Pago makes up only .677%, and the impacted land is .002%. Agricultural land is located on the southwest side of the island, where the topography is generally flat and at a greater distance.

North Carolina Landslide Causation from Rainfall

Problem

Western North Carolina is home to mountains, including the Great Smokey Mountain National Park. Swain County, NC, experiences greater landslides frequency from hurricanes, and large amounts of data require cleaning to be geospatially compatible for analysis.

Analysis Procedures

Taking a break from Python and Esri, I wanted to experience open-source products for the Extract Transform Load (ETL) process and analysis. I used R to connect and download landslide data from NC-One Mapserver, clean it, and transform it into spatial features. NOAA Rain gauge tabular data were imported and joined to weather stations.

Results

Once R coding prepared data, the maps were displayed in R ggplot, ArcGIS Pro, and QGIS. Hydrology studies also created flood rasters with ArcHydro, SAGA, and GRASS GIS toolbox plugins. Data files were complete and ready for transfer to the NC landslide department working with USDA and USGS. from the beach.

Web GIS

Air and Ground collection using UAVs and WebAppbuilder

Problem

During segregation, historic black communities were built in undesirable land plots such as Raleigh's 1954 southeast neighborhood. Urban sprawl has now made this community prone to flooding for years, and FEMA is ready to use a buyout program for specific homes. At the same time, the state looks at preserving the rest through landscape architecture techniques requiring GIS collection and analysis.

Analysis Procedures

Field assessment teams were split into riparian, greenway, and cemetery locations to collect observations using ArcCollector. Custom vector basemaps combined multiple features as aid references, drastically improving load time. At the cemetery, collectors collected grids, images, and plot numbers. Before collection, I taught students UAV techniques, collected aerial imagery for maps, and updated DEMs. New feature classes were built from a collection that linked tabular information and renamed images.

Results

Students were able to design analysis studies and storymaps showcasing landscape architecture techniques. I used the data to generate R charts with ggplot embedded into Esri StoryMaps, along with analysis from the DEM showing where the most damaged historic grave sites are located. Our studies from NCSU resulted in the City of Raleigh reversing a non-maintenance policy on private lands.
