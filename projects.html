Designing a GIS portfolio to showcase project work

Idea

When I graduated from NC State University, I had two GIS portfolios from GIS520 and the cumulative degree program GIS660. Those portfolio sites provided access to story maps and other web apps built during grad school. Showcasing your work is an excellent way to market yourself, and it has become a requirement for many GIS jobs. The previous portfolios helped me land my job at Esri, and while there, I shut it down when it was time to renew. Returning to the job market, I needed a new portfolio, and instead of a standard drag-and-drop Wix page, I wanted to code it from the ground up to improve my front-end coding skills.

Research

My previous experience in web page design was minimal. My previous portfolio was built using WordPress and a paid version of Elementor Pro. While at Esri, I built various cards, headers, and footers with Bootstrap for Hub Sites. In dashboards, I would build HTML tablesâ€”all of it on the fly, without really having a grasp of the code. I purchased UDEMY courses to learn HTML, CSS, Bootstrap5, and Javascript, which serve as the foundation of my code. I dipped my toes in NodeJS and SASS but did not implement them here. I had hoped to study Angular and React, but the initial studies took longer than expected.

Design

During the research, multiple concept sketches formed the basis of what I wanted to do and continued to change over time. The overall vision was an about-me page, followed by a series of sidebar pages covering education, employment, projects, and a blog. Pages for employment and blogs are not required, but I made them more for younger people who might come across my portfolio to serve as a road map or for transitioning veterans.

Methodology

The first code set began as Bootstrap and SASS. However, I was not ready to move on from HTML-CSS development, and I constantly forget to reinitiate the NodeJS SASS watch when opening up VScode. So, I completely dropped Bootstrap during version-01 to grasp the Grid and the Flexbox schema. For every HTML page, a corresponding CSS page helps reduce the amount of code per page. There are only a few code comments here, which I would never pass on to another person as I would need to refine the syntax (a lot), but version-01 is a get-out-the-door website. The webpage was designed on a 17" laptop, is not 508 complaint, nor is it mobile ready. You may need to adjust your screen size.

 

Version-02 will use Bootstrap as the core along with SASS and NodeJS, or I may skip "GO" and head directly to R-Shiny. Both Bootstrap and R-shiny have pros and cons. Bootstrap easily embeds a lot of the CSS and Java functionality directly into the HTML code, thus reducing time requirements. With R-shiny, you can incorporate interactive tables, maps, and charts.

Results

Multiple attempts at writing Java for listening events came up short when incorporating this time. For instance, you'll see that my expansion tab in the sidebar doesn't work right, or the images don't expand beyond their container. Many of these easy tasks are already baked into Bootstrap so I did not want to spend a great deal of time coming up with the solution. Overall the project was successful; meeting my goals of improving UI/UX design, programming, and displaying a portfolio for future employers.

Conclusion

Coding a website from the ground up is a specialty, just like a GIS analyst. People have four-year degrees to build web pages, so I would not recommend it to the average GIS person. I have a newfound appreciation for Esri Hub and Esri Experience Builder. Those Esri applications are so easy to use, and I 100% endorse any GIS student building a portfolio using either. The best thing about building in Esri is the capability of copying the back end through AGO Assistant so you can port the code into your next GIS position. My last sales pitch for using Esri Experience Builder is it's ability to design for mobile applications by adjusting the screen size and then moving your widgets to best fit.

About Portfolio

This website was built with pure HTML5, CSS, and Javascript. Previous versions included Bootstrap5, nodeJS, and SASS. Future upgraqdes will incorporate Leaflet and ChartJS. My portfolio was a culmination of developer courses taken at Udemy.
